# airtable_save_leads

## Description
Takes an array of leads (such as those generated by the `scrape_google_maps` workflow) and uploads them to an Airtable Base.

## Inputs
- `leads` (Array of Objects): The list of leads to save.
  - Each object must contain:
    - `name` (string): Name of the business.
    - `service` (string): The category of the business.
    - `address` (string): Physical address.
    - `website` (string): URL of the business.
    - `rating` (string): Review score.
    - `date_created` (string): Date scraped.
    - `status` (string): Lead status (e.g. "lead").

## Outputs
- `success` (boolean): `true` if all leads were successfully inserted, `false` otherwise.
- `inserted` (integer): Number of leads successfully added to Airtable.
- `errors` (Array context): Details of any failures.

## Configuration (Environment Variables)
- `AIRTABLE_PERSONAL_ACCESS_TOKEN`: The PAT used to authenticate with Airtable.
- `AIRTABLE_BASE_ID`: The ID of the Base where leads are stored (e.g. `app12345XYZ`).
- `AIRTABLE_TABLE_NAME`: The name of the table to insert into (e.g. `Leads`).

## Instructions for Airtable Setup
1. **Create an Airtable Base:** Make a new Base.
2. **Create a Table:** Name it `Leads` (or similar).
3. **Configure Columns EXACTLY as follows:**
   - `name` (Single line text)
   - `service` (Single line text)
   - `address` (Single line text)
   - `website` (URL)
   - `rating` (Single line text or Number)
   - `date_created` (Date)
   - `status` (Single select or Single line text)
4. **Generate a Personal Access Token:**
   - Go to [Airtable Developer Hub -> Personal Access Tokens](https://airtable.com/create/tokens).
   - Click "Create new token".
   - Name it "Telegram Lead Bot".
   - Under **Scopes**, add `data.records:write` and `data.records:read`.
   - Under **Access**, assign it to the Workspace or specific Base you created.
   - Click "Create token". Copy this value; it will start with `pat`.
5. **Get Base ID:**
   - Open your Airtable base in the browser. The URL looks like: `https://airtable.com/appXXXXXXXXXXXXXX/tbl...`
   - The string starting with `app` is your Base ID.
6. **Update `.env`**:
   Add the following to your `.env` file:
   ```env
   AIRTABLE_PERSONAL_ACCESS_TOKEN=patYourSecretTokenHere
   AIRTABLE_BASE_ID=appYourBaseIDHere
   AIRTABLE_TABLE_NAME=Leads
   ```
